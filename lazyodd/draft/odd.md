## 1. Purpose and Patterns

### Purpose

The purpose of SpatialRust is to explore how the interaction between shade management, weather conditions (temperature, rainfall, wind), and fungicide strategies affects Coffee Leaf Rust (*Hemileia vastarix*) epidemic dynamics and coffee farm productivity. {INFERRED} [inference: inferred from code structure — shade has complex competing effects on microclimate, dispersal, and photosynthesis; the parameter sweep tests temperature × rain × wind combinations]

The higher-level purpose is explanation and prediction: understanding the mechanisms by which shade trees mediate rust epidemics — simultaneously reducing sunlight (lowering photosynthesis), cooling local temperature, modifying rain splash dispersal distance, and blocking wind dispersal — and evaluating management strategies under varying weather scenarios. {INFERRED} [inference: shade mechanisms are central to the model architecture and affect nearly every submodel]

The model tracks three primary output metrics: disease incidence (fraction of plants with active lesions), disease severity (mean total lesion area per plant), and farm production (cumulative yield). {CODE_VERIFIED} [source: src/Metrics.jl:3-7]

### Patterns

The following patterns, drawn from empirical literature referenced in the model code, are encoded in the model and serve as criteria for evaluating its suitability:

1. **Lesion senescence**: Rust lesions senesce after approximately 5 months (~150 days), consistent with observations by McCain and Hennen (1984). {CODE_VERIFIED} [source: src/ABM/RustGrowth.jl:5-6]

2. **Rain wash-off of deposited spores**: Rain events wash spores off leaf surfaces, with a probability scaled by sunlight exposure, following Avelino et al. (2020). {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:65]

3. **Enhanced rain splash kinetic energy under shade**: Shade trees amplify the kinetic energy of throughfall, increasing rain splash dispersal distances. This is parameterized via `diff_splash`, based on findings by Avelino et al. (2020) that kinetic energy was approximately twice as high under shade, and Gagliardi et al. (2021) relating turbulent kinetic energy to canopy openness. {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:77]

4. **Deposited spore viability loss**: Ungerminated spores lose viability daily, with a retention rate based on Nutman et al. (1963). {CODE_VERIFIED} [source: src/ABM/RustGrowth.jl:194]

5. **Management incidence threshold**: A 10% observed incidence threshold triggers reactive fungicide application, following recommendations from Cenicafé's Boletín 36. {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:86]

6. **Calibration against empirical data**: Fourteen rust-related parameters were calibrated using Approximate Bayesian Computation (ABC), as indicated by the `p_row` parameter described as "parameter combination number (for ABC)". {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:37-38] The specific empirical datasets used for calibration are not documented in the source code. {UNVERIFIABLE} [reason: no calibration data or documentation available]

Note: The model's purpose and target patterns were inferred from the source code and parameter sweep structure, as no separate documentation describing the model's scientific objectives was available.

## 3. Process Overview and Scheduling

Each time step (one day), the function `step_model!` executes five processes in **fixed sequential order**. All processes are synchronous. The model does not use the Agents.jl built-in scheduler; instead, `step_model!` manually iterates over the agent vector. {CODE_VERIFIED} [source: src/ABM/MainStep.jl:3-9]

### Daily Schedule

```
step_model!(model):
  1. pre_step!(model)
  2. grow_shades!(model.current, shade_g_rate, max_shade)
  3. coffee_step!(model)
  4. rust_step!(model)
  5. farmer_step!(model)
```

**Process 1: pre_step!** — Updates day and tick counters, reads the current day's weather from the pre-generated `Weather` vectors (indexed by `ticks`), decays the outpour spore pool by a factor of 0.9, and if the day is windy, randomizes the wind heading from $\text{Uniform}(0, 360°)$. {CODE_VERIFIED} [source: src/ABM/MainStep.jl:14-35]

**Process 2: grow_shades!** — Applies discrete logistic growth to the global `ind_shade` variable (see Submodel 7.2). {CODE_VERIFIED} [source: src/ABM/ShadeSteps.jl:1-3]

**Process 3: coffee_step!** — Iterates over all coffee agents in deterministic storage order and applies the appropriate growth function based on the current phenological phase. The day within the annual cycle is determined by `mod1(days, 365)`: {CODE_VERIFIED} [source: src/ABM/MainStep.jl:37-84]

- Days `veg_d` to `rep_d - 1` (default 1–134): vegetative growth (Submodel 7.4)
- Day `rep_d` (default 135): vegetative growth followed by resource commitment to production (Submodel 7.6)
- Days `rep_d + 1` to 365 (default 136–365): reproductive growth (Submodel 7.5)

When `veg_d > rep_d` (non-default configurations), the code handles the wrap-around by negating all three values before comparison. For exhausted plants (`exh_countdown > 0`): if the countdown exceeds 1, it is decremented; if it equals 1, the plant regrows (Submodel 7.7).

**Process 4: rust_step!** — Selects function variants from eight combinations of three independent conditions: {fungicide active?, rain?, wind?}. This determines which germination function (`r_germinate!` vs. `nr_germinate!`), growth function (`grow_rust!` vs. `grow_f_rust!`), and dispersal functions (active vs. dummy) are used. The process then executes two loops: {CODE_VERIFIED} [source: src/ABM/MainStep.jl:86-164]

- **Loop 1**: For each agent where `rusted == true` (via a lazy `Iterators.filter`):
  1. **Disperse first**: If any lesion is sporulating, compute effective spore area and execute rain splash dispersal then wind dispersal
  2. Compute local temperature: $T_{\text{local}} = T - \text{temp\_cooling} \times (1 - \text{sunlight})$
  3. If `n_lesions > 0`: grow and sporulate lesions → check numerical bounds → germinate deposited spores → parasitize host
  4. If `n_lesions == 0` (only deposited spores): germinate only

- **Loop 2**: The lazy filter is **re-evaluated**, meaning plants newly marked `rusted = true` during Loop 1's dispersal are now included. For each rusted agent: execute `update_rust!` to merge new deposits into the deposited pool with viability loss.

- If windy: execute `outside_spores!` to re-inject spores from the outpour pool via the upwind farm edges.

**Critical design detail**: The separation of `newdeps` (spores arriving this step) from `deposited` (accumulated ungerminated spores) ensures that spores deposited during the current step do not participate in germination until the following step. Dispersal adds to `newdeps`; germination reads only from `deposited`; `update_rust!` in Loop 2 merges them: $\text{deposited} \leftarrow \text{deposited} \times \text{viab\_loss} + \text{newdeps}$. {CODE_VERIFIED} [source: src/ABM/RustGrowth.jl:190-204]

**Process 5: farmer_step!** — Executes management actions based on the current day of year (`mod1(days, 365)`): {CODE_VERIFIED} [source: src/ABM/MainStep.jl:170-206]

1. If harvest day: execute full harvest cycle (Submodel 7.17)
2. If prune day (matches `prune_sch`): prune shades (Submodel 7.18)
3. If inspection day (`days % inspect_period == 0`): inspect plants (Submodel 7.19)
4. Fungicide logic:
   - If fungicide is active (`fungicide > 0`): increment counter; expire when counter exceeds 30 (hardcoded)
   - If inactive, not harvest day, and under maximum annual sprayings: apply fungicide based on the selected strategy (Submodel 7.20)
5. Accumulate `shadeacc += ind_shade` for annual shade averaging

### Agent Execution Order

Agents are stored in a `Vector{Coffee}` and iterated in the order they were added. Agents are added by scanning the `farm_map` via `findall(x -> x == 1, farm_map)`, which returns positions in column-major order (Julia's default array traversal). This order is deterministic and fixed for the duration of the simulation. No randomization of agent execution order occurs. {CODE_VERIFIED} [source: src/ABM/CreateABM.jl:17-38, src/SpatialRust.jl:8-11]

### Warm-Up Period

Before the main simulation, a 2-year warm-up (`pre_run365!`) establishes quasi-equilibrium states: {CODE_VERIFIED} [source: src/ABM/CreateABM.jl:112-200]

- Runs for 730 days executing only `grow_shades!`, `coffee_step!`, and explicitly scheduled `prune_shades!` calls
- Does NOT execute `pre_step!` (no weather reading, no tick increment), `rust_step!`, or `farmer_step!`
- Simplified harvest at days 365 and 730: only resets `production = 0.0` per plant (no lesion survival logic, no cost accounting)
- After completion: resets `days = 0` and `costs = 0`

## 4. Design Concepts

### 4.1 Basic Principles

The model integrates three theoretical domains: {INFERRED} [inference: inferred from code structure; no explicit theoretical documentation available]

1. **Plant physiology**: Carbon assimilation is modeled using Michaelis-Menten (Monod) kinetics with dual saturating responses to light and photosynthetic tissue, followed by allocation tradeoffs between vegetative growth, storage, and fruit production.

2. **Epidemiology**: The complete Coffee Leaf Rust lifecycle is represented, including spore deposition, germination and infection (temperature- and moisture-dependent), lesion growth, sporulation, rain splash and wind dispersal, parasitism of host resources, and lesion senescence.

3. **Agroecology**: Shade trees serve as mediators that simultaneously affect microclimate (temperature cooling), light availability (photosynthesis reduction), and physical barriers to dispersal (blocking wind-dispersed spores, modifying rain splash kinetics).

An agent-based approach was chosen because the model requires spatial heterogeneity in shade cover, directional dispersal (wind heading), individual plant state tracking (storage depletion leading to exhaustion), and explicit farm layout geometry — all of which necessitate individual-level representation.

### 4.2 Emergence

Epidemic dynamics (incidence and severity curves over time), spatial patterns of infection (driven by dispersal mechanics and farm layout), and production losses (from parasitism draining individual plant storage) all emerge from individual-level processes. {INFERRED} [inference: these system-level outcomes are not directly imposed but arise from agent interactions]

The following are imposed and do not emerge: weather (pre-generated time series), farm layout (fixed at initialization), and management schedules (calendar-based pruning and fungicide timing). {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:232-236]

### 4.3 Adaptation

Coffee plants do not adapt their behavior to changing conditions. The farmer exhibits limited reactive adaptation through incidence-triggered fungicide strategies: under the `:incd` and `:cal_incd` strategies, fungicide is applied when the observed incidence (from the most recent inspection) exceeds 10%, a threshold drawn from Cenicafé's Boletín 36. This behavior is reactive — responding to current observed conditions — not optimizing. {CODE_VERIFIED} [source: src/ABM/MainStep.jl:186-201, src/ABM/MainSetup.jl:86]

### 4.4 Objectives

Not applicable. No agents pursue explicit objectives or attempt to optimize any measure. The farmer follows rule-based management strategies without utility maximization. {CODE_VERIFIED}

### 4.5 Learning

Not applicable. No agents change their decision rules or adaptive traits over time based on experience. Commented-out code references a genetic algorithm (GA) for strategy evolution, suggesting this capability was explored but is not active in the current version. {CODE_VERIFIED} [source: src/ABM/MainStep.jl:31-33]

### 4.6 Prediction

Not applicable. No agents estimate or anticipate future conditions when making decisions. Fungicide application is either calendar-based or reactive to currently observed incidence. {CODE_VERIFIED}

### 4.7 Sensing

**Coffee plants** sense their local sunlight level, computed from their position-specific `shade_map` value scaled by the current global `ind_shade` level: $\text{sunlight} = 1 - \text{shade\_map}[\text{pos}] \times \text{ind\_shade}$. This is the only information they access. {CODE_VERIFIED} [source: src/ABM/CoffeeSteps.jl:2-3]

**Rust processes** are implicitly affected by local temperature (reduced by shade cooling: $T_{\text{local}} = T - \text{temp\_cooling} \times (1 - \text{sunlight})$) and by current weather (rain, wind). This is not modeled as deliberate agent sensing. {CODE_VERIFIED} [source: src/ABM/MainStep.jl:141]

**The farmer** senses disease status through periodic inspection. Every `inspect_period` days (default 7), a sample of `inspect_effort` (default 1%) of active coffee plants is randomly selected and inspected. Sensing is imperfect: only lesions with area > 0.05 (approximately 0.25 cm diameter) are visible, and detection probability depends on lesion size — a plant is detected as infected if the maximum lesion area exceeds 0.8, or otherwise with probability $n_{\text{visible}} / 5$. The resulting `obs_incidence` is updated only at inspection intervals, not continuously. {CODE_VERIFIED} [source: src/ABM/CGrowerSteps.jl:45-75]

### 4.8 Interaction

**Rust-mediated indirect interaction**: Infected coffee plants disperse spores to other plants via two mechanisms: rain splash dispersal (local, omnidirectional, mean distance ~2.2 cells) and wind dispersal (directional, following wind heading ± 15°, mean distance ~5.8 cells). This is the primary interaction pathway in the model. {CODE_VERIFIED} [source: src/ABM/RustDispersal.jl:3-52]

**Shade-mediated indirect interaction**: Shade trees affect all coffee plants within their radius of influence (`shade_r = 3` cells) through the static `shade_map`. The shade level is spatially heterogeneous (based on distance to nearest shade tree) and temporally dynamic (scaled by the global `ind_shade` variable, which grows logistically and is reduced by pruning). {CODE_VERIFIED} [source: src/ABM/ShadeMap.jl:9-33]

**No direct agent-agent interaction**: Coffee plants do not communicate with or directly affect each other. There is no density dependence between plants (spacing is fixed by the farm map).

**Farm boundary interaction**: Spores that exit the farm boundary enter the `outpour` pool, a vector tracking spore counts in eight compass directions. This pool decays by a factor of 0.9 daily. On windy days, spores from the outpour pool re-enter the farm from upwind edges, creating an attenuated directional recycling effect. {CODE_VERIFIED} [source: src/ABM/RustDispersal.jl:160-267]

## 5. Initialization

### Farm Layout

The spatial arrangement of coffee plants and shade trees is defined by the `farm_map`, an integer matrix of size `map_side` × `map_side` (default 100 × 100) where 0 = empty cell, 1 = coffee plant, and 2 = shade tree. Three layout options are available, controlled by the `common_map` parameter: {CODE_VERIFIED} [source: src/ABM/FarmMap.jl:2-65, src/ABM/MainSetup.jl:140-151]

1. **Custom layout** (`:none`, default): Coffee plants placed in rows with spacing `row_d` (default 2) between rows and `plant_d` (default 1) between plants within rows. Shade trees added in a regular pattern with spacing `shade_d` (default 6) or randomly. Optional barrier rows of shade trees (`barrier_rows = 2`, `barriers = (1, 0)` = one internal barrier set, no edge barriers).

2. **Full sun** (`:fullsun`): Coffee plants only, no shade trees.

3. **Regular shaded** (`:regshaded`): Coffee plants with regularly spaced shade trees at interval `shade_d`.

A user-supplied `farm_map` can replace the generated layout, overriding `map_side` to match its dimensions. {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:148-149]

### Shade Map

A static `shade_map` matrix is computed once at initialization. For each cell, the shade value is calculated as $1.0 - d_{\min}^2 / (2 \times \text{shade\_r})^2$, where $d_{\min}^2$ is the squared Euclidean distance to the nearest shade tree within radius `shade_r` (default 3 cells). Shade tree cells receive a value of 1.0. Cells with no shade tree within the radius receive 0.0. For full-sun layouts, the shade map is all zeros. {CODE_VERIFIED} [source: src/ABM/ShadeMap.jl:9-35]

### Coffee Agent Placement

One coffee agent is created for each cell where `farm_map == 1`, placed in column-major order (Julia's default `findall` traversal). Initial state variable values: {CODE_VERIFIED} [source: src/ABM/CreateABM.jl:17-38]

- `veg = 2.0`
- `storage` = $75.5 \times e^{-5.5 \times \text{sunlight}} + 2.2$ (inverse relationship: shadier positions start with higher storage)
- `production = 0.0`
- `rust_gr` drawn from $\text{Truncated}(\mathcal{N}(0.0867, 0.000867), 0, 0.13)$ — providing individual heterogeneity in susceptibility to rust
- `sunlight` computed from the shade map and initial `ind_shade`
- All rust state variables initialized to zero/empty/false

### Warm-Up

A 2-year (730-day) warm-up period (`pre_run365!`) is executed before the main simulation. During warm-up: {CODE_VERIFIED} [source: src/ABM/CreateABM.jl:112-200]

- Only `grow_shades!` (shade logistic growth), `coffee_step!` (full coffee phenology), and explicitly scheduled `prune_shades!` are executed
- No weather is read (`pre_step!` is not called), no rust processes occur, and no farmer actions beyond pruning are taken
- A simplified harvest occurs at days 365 and 730, resetting only `production = 0.0` per plant
- After completion, `days` and `costs` are reset to 0

The warm-up allows coffee biomass variables (`veg`, `storage`) and the shade level (`ind_shade`) to reach approximate equilibrium before rust is introduced.

### Rust Inoculation

After the warm-up, rust is introduced according to the `p_rusts` parameter (default 0.01): {CODE_VERIFIED} [source: src/ABM/CreateABM.jl:52-110]

- If $0 < \text{p\_rusts} < 1$: A fraction `p_rusts` of all coffee agents are randomly selected (uniform, without replacement) and inoculated
- If $\text{p\_rusts} = 1$: One spatial cluster is created — a random center (excluding grid margins) with all coffee plants within Chebyshev radius 1 inoculated
- If $\text{p\_rusts} > 1$: Multiple clusters, one per integer value

Each inoculated plant receives $1 + \text{Binomial}(\text{max\_lesions} - 1, 0.05)$ lesions. Each lesion is assigned a random initial area from $\text{Uniform}(0, 0.3)$, then categorized:

- Area < 0.01: counted as a deposited spore (increments `deposited`; not added as a lesion)
- $0.01 \leq \text{area} < 0.28$: non-sporulating lesion (age derived from area × 100)
- Area ≥ 0.28: sporulating lesion

### Weather

Daily weather is pre-generated for the entire simulation length at initialization. If external data vectors are provided (`rain_data`, `wind_data`, `temp_data`), noise is added: 5% bit-flip for boolean vectors, $\mathcal{N}(0, 0.05)$ for temperature. If not provided, weather is generated stochastically from probability parameters (see Submodel 7.1). {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:362-383]

### Variation Between Runs

Initialization varies between simulation runs due to stochastic elements: weather time series, individual `rust_gr` values, initial rust placement, and management cost noise. Setting `seed > 0` in `init_spatialrust` initializes the random number generator (`Xoshiro(seed)`) to produce fully reproducible runs. {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:136]

## 7. Submodels

### 7.1 Weather Generation

**Purpose.** Generate or process daily weather time series for the full simulation.

**Logic.**

```
FOR t = 1 TO steps:
  IF rain_data provided:
    rain[t] ← rain_data[t] with 5% bit-flip noise
  ELSE:
    rain[t] ~ Bernoulli(rain_prob)

  IF wind_data provided:
    wind[t] ← wind_data[t] with 5% bit-flip noise
  ELSE:
    IF rain[t]:
      wind[t] ~ Bernoulli(wind_prob − 0.1)
    ELSE:
      wind[t] ~ Bernoulli(wind_prob + 0.1)

  IF temp_data provided:
    temp[t] ← temp_data[t] + N(0, 0.05)
  ELSE:
    temp[t] ~ N(mean_temp, 0.5)
```

Wind probability is conditionally adjusted: lower during rain (−0.1), higher during dry conditions (+0.1), creating a negative correlation between simultaneous rain and wind events. {CODE_VERIFIED} [source: src/ABM/MainSetup.jl:352-383]

### 7.2 Shade Tree Growth

**Purpose.** Model shade canopy recovery between pruning events via logistic growth of the global shade level.

**Equation.**

### 7.3 Coffee Photosynthesis

**Purpose.** Compute daily gross carbon assimilation for a coffee plant. This submodel is shared by vegetative growth (7.4) and reproductive growth (7.5).

**Equations.**

$$\text{photo\_veg} = \text{veg} \times \text{photo\_frac}$$

$$\text{PhS} = \text{photo\_const} \times \frac{\text{sunlight}}{\text{k\_sl} + \text{sunlight}} \times \frac{\text{photo\_veg}}{\text{k\_v} + \text{photo\_veg}}$$

where $\text{photo\_const} = \text{f\_avail} \times \text{phs\_max}$ (precomputed at initialization). The equation is a product of two Michaelis-Menten (Monod) saturating response functions: one for light availability (`sunlight`) and one for photosynthetic tissue quantity (`photo_veg`). {CODE_VERIFIED} [source: src/ABM/CoffeeSteps.jl:6-8]

| Light half-saturation | `k_sl` | 0.05 | dimensionless | Half-saturation constant for sunlight response | Model parameter |

### 7.4 Vegetative Growth

**Purpose.** Allocate photosynthate to vegetative biomass and storage during the vegetative phase (default days 1–134).

**Equations.**

$$\text{veg} \leftarrow \text{veg} + \text{phs\_veg} \times \text{PhS} - \mu_{\text{veg}} \times \text{veg}$$

$$\text{storage} \leftarrow \text{storage} + \text{phs\_sto} \times \text{PhS}$$

$$\text{veg} \leftarrow \max(\text{veg}, 0)$$

Note: $\text{phs\_veg} + \text{phs\_sto} = 0.8 + 0.6 = 1.4 > 1.0$ because these represent conversion efficiencies of different allocation streams, not fractions of the same pool. {CODE_VERIFIED} [source: src/ABM/CoffeeSteps.jl:6-15]

### 7.5 Reproductive Growth

**Purpose.** Allocate photosynthate during the reproductive phase (default days 136–365), balancing vegetative maintenance, fruit production growth, and storage management.

**Logic.** The allocation regime depends on the plant's storage status.

```
frac_v ← veg / (veg + production)

IF storage < 0 (stressed plant):
  Δprod ← PhS − μ_prod × production
  IF Δprod > 0:
    remainder ← Δprod − μ_veg × veg
    veg ← veg + min(remainder, 0)
    storage ← storage + max(remainder, 0)
  ELSE:
    veg ← veg − μ_veg × veg
    production ← production + Δprod

ELSE (storage ≥ 0, healthy plant):
  d_prod ← (1 − frac_v) × PhS − μ_prod × production
  IF d_prod < 0 (production declining):
    veg ← veg + phs_veg × frac_v × PhS − μ_veg × veg
    storage ← storage + 0.95 × d_prod
    production ← production + 0.05 × d_prod
  ELSE (d_prod ≥ 0, production growing):
    veg ← veg + phs_veg × (d_prod + frac_v × PhS) − μ_veg × veg

veg ← max(veg, 0)
production ← max(production, 0)
```

Under stress (storage depleted), the plant prioritizes maintaining production from photosynthate, with any surplus recovering vegetative biomass and then storage. Under healthy conditions, allocation between vegetation and production is proportional to their relative biomass, and when production declines, 95% of the loss is absorbed by storage with only 5% as actual production loss. {CODE_VERIFIED} [source: src/ABM/CoffeeSteps.jl:17-55]

### 7.8 Rust Lesion Growth (No Fungicide)

**Purpose.** Daily aging, senescence, growth, and sporulation of rust lesions when no fungicide is active.

**Logic.**

```
STEP 1 — Senescence:
  Remove all lesions with age > 150 days

STEP 2 — Aging:
  FOR each lesion: age ← age + 1

STEP 3 — Temperature modifier:
  temp_mod ← −(1 / temp_ampl²) × (local_temp − opt_temp)² + 1.0 − 0.1 × sunlight
  IF temp_mod ≤ 0: EXIT (no growth or sporulation this day)

STEP 4 — Sporulation (non-sporulating lesions only):
  spor_mod ← temp_mod × rain_spo × (1 + rep_spo × production / (production + veg))
  FOR each lesion i WHERE spores[i] = false:
    spores[i] ← Bernoulli(areas[i] × spor_mod)

STEP 5 — Area growth:
  host_gro ← 1 + rep_gro × production / max(storage, 1.0)
  growth_mod ← rust_gr × temp_mod × host_gro
  area_gro ← max(0, 1 − sum(areas) / 25.0)
  FOR each lesion i:
    areas[i] ← areas[i] + areas[i] × growth_mod × area_gro
```

where `rain_spo` = 1.0 if rain, `pdry_spo` if dry; and $\text{local\_temp} = \text{temperature} - \text{temp\_cooling} \times (1 - \text{sunlight})$.

The temperature modifier is a downward parabola centered at `opt_temp`, with the term $-0.1 \times \text{sunlight}$ representing UV inhibition of rust under direct sunlight.

**Notable implementation detail**: The $-0.1 \times \text{sunlight}$ UV inhibition term is present in `grow_rust!` (no fungicide) but **absent** from `grow_f_rust!` (with fungicide). This asymmetry exists in the source code. {CODE_VERIFIED} [source: src/ABM/RustGrowth.jl:4-35]

The `area_gro` term implements density-dependent growth limitation: as total lesion area approaches 25.0, growth slows to zero. The `host_gro` term increases rust growth when the plant has higher production relative to storage, representing the effect of the host's reproductive state on rust vigor.

Senescence at 150 days follows observations by McCain and Hennen (1984). {CODE_VERIFIED} [source: src/ABM/RustGrowth.jl:5]

| Parameter | Symbol | Value | Units | Description | Source |
|-----------|--------|-------|-------|-------------|--------|
| Base growth rate | `rust_gr` | 0.0867 | day⁻¹ | Per-plant (individual heterogeneity) | ABC calibration |
| Optimal temperature | `opt_temp` | 22.8716 | °C | Temperature for maximum rust growth | ABC calibration |
| Temperature amplitude | `temp_ampl` | 5.0367 | °C | Width of temperature response curve | ABC calibration |

### 7.9 Rust Lesion Growth (With Fungicide)

**Purpose.** Same as Submodel 7.8 but with fungicide effects modifying growth, sporulation, and lesion survival.

**Differences from Submodel 7.8:**

1. **No UV inhibition**: The temperature modifier omits the $-0.1 \times \text{sunlight}$ term:
   $\text{temp\_mod} = -(1/\text{temp\_ampl}^2) \times (\text{local\_temp} - \text{opt\_temp})^2 + 1.0$

2. **Per-lesion preventive/curative classification**:
   ```
   FOR each lesion i:
     prev_cur[i] ← (ages[i] − fday) < 15
   ```
   Where `fday` is the current fungicide counter. If true, the lesion was younger than 15 days when fungicide was applied (preventive effect); otherwise, curative effect.

3. **Modified sporulation**: $\text{spor\_probs}[i] = \text{spor\_mod} \times \begin{cases} \text{fung\_spor\_prev} & \text{if prev\_cur}[i] \\ \text{fung\_spor\_cur} & \text{otherwise}\end{cases}$

4. **Modified growth**: $\text{gro\_mods}[i] = \text{growth\_mod} \times \begin{cases} \text{fung\_gro\_prev} & \text{if prev\_cur}[i] \\ \text{fung\_gro\_cur} & \text{otherwise}\end{cases}$

5. **Active area reduction** (first 15 days after application):
   ```
   IF fday < 15:
     areas ← areas × fung_reduce
     Remove lesions where area < 0.00005
   ```

### 7.10 Germination and Infection (Rain)

**Purpose.** Deposited spores attempt to germinate and penetrate leaf tissue on rainy days.

**Logic.** For each deposited spore (iterating while $sp \leq \text{deposited}$):

```
STEP 1 — UV inhibition:
  IF Bernoulli(sunlight × light_inh):
    deposited ← deposited − 1
    CONTINUE

STEP 2 — Rain wash-off:
  IF Bernoulli(sunlight × rain_washoff):
    deposited ← deposited − 1
    CONTINUE

STEP 3 — Infection attempt (if n_lesions < max_lesions):
  temp_inf_p ← 1 − 0.0137457 × (local_temp − 21.5)²
  wet_inf_p ← 0.05 × (18.0 + 2.0 × sunlight) − 0.2
  host ← 1 − rep_inf + rep_inf × production / (production + veg)
  infection_p ← max_inf × temp_inf_p × wet_inf_p × host × fung_modifier

  IF Bernoulli(infection_p):
    deposited ← deposited − 1
    n_lesions ← n_lesions + 1
    Push new lesion: age = 0, area = 0.00005, spores = false
```

The `wet_inf_p` term represents leaf wetness duration: a base of 18 hours with rain plus a small sunlight-dependent adjustment. The coefficient 0.0137457 produces a temperature-dependent infection probability centered at 21.5°C. The `host` term modulates infection probability by the plant's reproductive state. When fungicide is active, `fung_modifier = fung_inf` (0.9); otherwise 1.0. {CODE_VERIFIED} [source: src/ABM/RustGrowth.jl:93-131]

### 7.14 Rain Splash Dispersal

**Purpose.** Local stochastic spore dispersal via rain droplet splash.

**Logic.**

```
Effective spore area:
  spore_area ← sum(areas[i] for sporulating lesions) × spore_pct × (1 + sunlight)

Number of splashed spores:
  n_splashed ~ Poisson(spore_area)

Distance modifier (U-shaped function of shade):
  d_mod ← (4 − 4 × diff_splash) × (sunlight − 0.5)² + diff_splash

FOR each of n_splashed spores:
  distance ~ Exponential(rain_dst) × d_mod
  heading ~ Uniform(0, 360°)

  IF distance < 1.0:
    Self-deposition: newdeps ← newdeps + 1

  ELSE:
    Trace trajectory from pos along heading in 0.5-cell increments:
      AT each new cell (newx, newy):
        IF outside farm boundary:
          outpour[direction] ← outpour[direction] + 1; STOP
        IF farm_map[newx, newy] = 1 (coffee) AND Bernoulli(tree_block):
          Deposit on that plant; STOP
        IF farm_map[newx, newy] = 2 (shade) AND Bernoulli(tree_block):
          Spore lost (absorbed); STOP
      IF reaches max distance without interception:
        IF final cell is coffee: deposit
        ELSE: spore lost
```

The `d_mod` function creates a U-shaped relationship: rain splash distance is maximized at both full sun (`sunlight = 1.0`) and full shade (`sunlight = 0.0`), and minimized at intermediate shade (`sunlight = 0.5`). This captures the finding that shade trees amplify throughfall kinetic energy (Avelino et al. 2020; Gagliardi et al. 2021). {CODE_VERIFIED} [source: src/ABM/RustDispersal.jl:3-26 (disperse_rain!); src/ABM/RustDispersal.jl:56-101 (splash trajectory)]

| Parameter | Symbol | Value | Units | Description | Source |
|-----------|--------|-------|-------|-------------|--------|
| Mean rain distance | `rain_dst` | 2.2071 | cells | Mean distance of Exponential dispersal kernel | ABC calibration |
| Splash distance modifier | `diff_splash` | 2.5 | multiplier | Shade-enhanced splash distance | Avelino et al. 2020; Gagliardi et al. 2021 |

### 7.15 Wind Dispersal

**Purpose.** Long-range directional spore dispersal via wind.

**Logic.**

```
Spores lifted:
  n_lifted ~ Poisson(spore_area × (1 − 0.5 × shade_map[pos]))

Dispersal distance:
  w_distance ~ Exponential(wind_dst) × (1 + sunlight × diff_wind)

IF w_distance < 1.0:
  Self-deposition: newdeps ← newdeps + n_lifted

ELSE:
  FOR each of n_lifted spores:
    heading ← wind_h + Uniform(0, 30°) − 15°
    Trace trajectory via gust() in 0.5-cell increments:
      AT each new cell:
        IF outside boundary: outpour[direction] += 1; STOP
        Blocking check: P(blocked) = shade_map[cell] × shade_block
        IF blocked:
          NEXT cell encountered:
            IF coffee: deposit; STOP
            ELSE: spore lost; STOP
      IF reaches max distance:
        IF final cell is coffee: deposit
        ELSE: spore lost
```

Key differences from rain splash dispersal:
- **Directional**: Follows wind heading ± 15° (not omnidirectional)
- **Shade blocks probabilistically**: Based on continuous `shade_map` value × `shade_block`, not just tree presence
- **Two-step blocking**: Once a spore is "blocked" (slowed by canopy), it deposits on the **next** cell encountered, not the blocking cell
- **Shade reduces liftoff**: Fewer spores are lifted from shaded positions ($1 - 0.5 \times \text{shade\_map}$)
- **Openness increases distance**: More sunlit positions disperse farther ($1 + \text{sunlight} \times \text{diff\_wind}$)

{CODE_VERIFIED} [source: src/ABM/RustDispersal.jl:28-52 (disperse_wind!); src/ABM/RustDispersal.jl:103-156 (gust trajectory)]

| Mean wind distance | `wind_dst` | 5.7806 | cells | Mean distance of Exponential dispersal kernel | ABC calibration |
| Wind distance modifier | `diff_wind` | 3.0 | multiplier | Openness-enhanced wind distance | Pezzopane |

### 7.16 Outside Spore Re-entry

**Purpose.** Spores that exited the farm boundary can return on windy days.

The `outpour` vector (8 elements) tracks spore counts that exited in each compass direction. This pool decays by a factor of 0.9 daily (in `pre_step!`). On windy days, `outside_spores!` selects the upwind directions based on `wind_h`:

- The wind heading is divided into 8 sectors of 45° each (cardinal and ordinal directions)
- For each relevant sector, the corresponding outpour pool spores attempt re-entry
- Starting positions are random points along the corresponding farm edge
- Each spore's trajectory is computed using `gust()` with distance $\sim \text{Exponential}(\text{wind\_dst}) \times (1 + \text{diff\_wind})$
- Spores that reach coffee plants deposit; others are lost

{CODE_VERIFIED} [source: src/ABM/RustDispersal.jl:160-309]

### 7.17 Harvest

**Purpose.** Annual harvest at the end of the production cycle, including cost accounting and rust population reduction.

**Logic.**

```
yearly_prod ← sum(production) over all agents
avg_shade ← shadeacc / 365.0
costs ← costs + fixed_costs + yearly_prod × (other_costs × (1 − avg_shade × mean(shade_map)) + 0.012)
shadeacc ← 0
fung_count ← 0

FOR each agent:
  production ← 0
  deposited ← deposited × lesion_survive
  surviving ← floor(n_lesions × lesion_survive)
  Remove the OLDEST (first) (n_lesions − surviving) lesions from ages, areas, spores
  n_lesions ← surviving
  IF surviving = 0 AND deposited < 0.05:
    deposited ← 0; rusted ← false
```

The cost calculation incorporates shade-dependent variable costs: higher shade levels reduce the `other_costs` component (representing reduced weeding under shade). The constant 0.012 represents a fixed per-unit-production cost. {CODE_VERIFIED} [source: src/ABM/CGrowerSteps.jl:1-34]

### 7.18 Shade Pruning

**Purpose.** Reduce the global shade level on scheduled pruning days.

**Logic.**

```
IF ind_shade > target:
  ind_shade ← target
ELSE:
  ind_shade ← ind_shade × 0.9
costs ← costs + tot_prune_cost
```

Where `tot_prune_cost` = $\mathcal{N}(\text{prune\_cost}, 0.05 \times \text{prune\_cost}) \times \text{n\_shades}$ (computed at initialization with noise).

When current shade exceeds the target, it is set directly to the target. When already below the target (e.g., after a previous pruning or slow recovery), shade is further reduced by 10%. {CODE_VERIFIED} [source: src/ABM/CGrowerSteps.jl:36-43]

### 7.19 Inspection

**Purpose.** The farmer samples plants, detects and removes visible lesions, and updates observed incidence.

**Logic.**

```
n_inspected ← floor(inspect_effort × n_coffees)
inspected ← sample(active plants, n_inspected, replace = false)

n_infected ← 0
FOR each plant in inspected:
  n_visible ← count(areas[i] > 0.05)
  IF n_visible > 0 AND (max(areas) > 0.8 OR Bernoulli(n_visible / 5)):
    n_infected ← n_infected + 1
    Select up to rm_lesions lesions weighted by visibility:
      weight[i] = areas[i] if areas[i] > 0.05, else 0
    Remove selected lesions from ages, areas, spores
    n_lesions ← n_lesions − (number removed)
    IF n_lesions = 0 AND deposited < 0.05:
      rusted ← false

obs_incidence ← n_infected / n_inspected
costs ← costs + tot_inspect_cost
```

Lesions must have area > 0.05 (approximately 0.25 cm diameter) to be visible. Detection is certain when the maximum lesion area exceeds 0.8 (~1 cm diameter); otherwise it occurs with probability $n_{\text{visible}} / 5$. Lesion removal is weighted by visibility (larger lesions are more likely to be removed). {CODE_VERIFIED} [source: src/ABM/CGrowerSteps.jl:45-75]

| Lesions removed | `rm_lesions` | 2 | count | Max lesions removed per infected plant | Model parameter |

### 7.20 Fungicide Application

**Purpose.** Apply fungicide spray to the entire farm.

**Logic.**

```
costs ← costs + tot_fung_cost
fungicide ← 1
fung_count ← fung_count + 1
```

Where `tot_fung_cost` = $\mathcal{N}(\text{fung\_cost}, 0.05 \times \text{fung\_cost}) \times \text{n\_coffees}$ (computed at initialization). Fungicide remains active for 30 days (the counter increments daily in `farmer_step!` and the effect expires when the counter exceeds 30). Maximum `max_fung_sprayings` (default 3) applications per year.

**Notable implementation detail**: The parameter `fung_effect` (default 30) is defined in `MngPars` but is not used in the code; the fungicide duration of 30 days is hardcoded in `farmer_step!`. {CODE_VERIFIED} [source: src/ABM/CGrowerSteps.jl:79-83]

**Four application strategies** (controlled by `fung_stratg`): {CODE_VERIFIED} [source: src/ABM/MainStep.jl:186-201]

1. **`:cal`** (calendar): Spray on days in `fungicide_sch` (default: days 91, 273)
2. **`:cal_incd`** (hybrid): Spray on scheduled days OR when `obs_incidence > incidence_thresh`
3. **`:incd`** (incidence-only): Spray only when `obs_incidence > incidence_thresh`
4. **`:flor`** (flowering-based): Spray at `rep_d + 60` and `rep_d + 105`; if incidence exceeds threshold after the second spray, additional applications follow the incidence strategy

| Strategy | `fung_stratg` | `:cal` | symbol | Application strategy | Model parameter |
| Incidence threshold | `incidence_thresh` | 0.1 | fraction | Threshold for reactive spraying | Cenicafé Boletín 36 |
| Max annual sprays | `max_fung_sprayings` | 3 | count/year | Maximum applications per year | Model parameter |
